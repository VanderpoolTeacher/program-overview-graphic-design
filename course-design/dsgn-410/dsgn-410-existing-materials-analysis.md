## Gaps the existing materials do not fill

1. **Data visualization (CLO 5).** No assignment asks students to produce a
   data-driven artifact. CLO 5 explicitly says "branded content, data
   visualizations, and presentations" — currently only branded content and
   presentations are exercised. Adding a data-viz component (e.g., visualize
   freelance income over time, market positioning chart, client-pipeline
   dashboard) would close this gap and is the only place in DSGN 410 where
   MLO 5 (Data Visualization) is exercised at the course level.

2. **Proposal writing (CLO 2).** CLO 2 names "budgets, **timelines**, and
   **proposals**." The Project Management Plan covers timelines and the
   Legal/Financial Responsibilities assignment touches budgets, but no
   assignment requires students to write a client proposal. Worth its own
   deliverable or a section added to the PM plan.

3. **AI as primary tool (CLO 6).** Every assignment ends with a 3–4
   sentence "did you use AI?" reflection. CLO 6 is more ambitious: students
   should *use* AI for estimating, drafting, client comms, market research.
   Consider an assignment where AI-driven workflow is the deliverable, not
   a footnote.

4. **Ethics scenarios (CLO 4).** Ethics is currently scattered. A dedicated
   ethics scenario assignment (e.g., what would you do when X happens?)
   would give CLO 4 a clear home and create a teachable moment.

5. **Tooling.** No assignment specifies tools — invoicing software,
   contract templates/platforms (e.g., HelloSign, AND.CO), AI tools, PM
   platforms (Notion, Asana, Trello), accounting (QuickBooks Self-Employed).
   Students will benefit from concrete recommendations or a curated toolkit.

6. **Internship integration.** DSGN 410 runs alongside DSGN 491
   (Internship). None of the assignments leverage the internship as a real
   client/project source. Several could ("draft a contract for a project
   you handled at your internship"; "create an invoice for the work you
   did this week").

## Mismatches and tensions

- **Credit hours.** The plan in `source/program-data.md` lists DSGN 410 as
  **1 credit** (Spring 2029). The 15-week schedule and the assignment load
  here are sized for a 3-credit course. Either the credit count needs to
  increase or the assignment count needs to decrease. Worth raising with
  the program before further content development.

- **Book selection up front.** The "Syllabus Review and Book Selection"
  assignment hands curriculum choice to students. This is great for
  ownership but means assignments cannot reliably reference common
  readings. Alternatives: (a) instructor selects a core text + students
  add 1 supplementary book; (b) keep as-is but build assignments to be
  text-agnostic.

- **Rubric inconsistency.** Five assignments use a 4-point competency
  rubric (Exemplary/Proficient/Developing/Beginning); the Syllabus Review
  uses a 5-point scale (Excellent/Good/etc., 20 pts total); the
  Self-Marketing assignment has no formal rubric, just "Assessment
  Criteria." Aligning to one rubric system would help students and
  graders.

- **CLO mapping not yet declared.** None of the assignments declare which
  CLOs they target. Adding a "CLOs addressed" header to each assignment
  (in the same format as `source/program-data.md`) would make the
  alignment auditable.

## Quick wins (high value, low effort)

- Add a "CLOs addressed" line to each existing assignment.
- Add a small data-visualization component to the Self-Marketing or
  Project Management assignment (e.g., a Gantt chart, freelance-rate
  comparison chart).
- Replace the trailing "AI reflection" boxes with one assignment-grade
  AI-workflow deliverable.
- Decide on credit hours and prune the schedule to match.
